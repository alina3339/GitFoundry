Common questions

The things people ask before they buy.

Is hosting really free, forever?

Hosting on GitHub Pages is free with no time limit — it's provided as part of GitHub (owned by Microsoft). You only pay for your domain, which is around £12 a year direct to a registrar like Namecheap or Porkbun. That money never comes to GitFoundry.

What if GitHub changes its terms?

Every site I deliver is portable. The handover packet includes a migration playbook showing exactly how to redeploy your repository to Cloudflare Pages, Netlify, GitLab Pages, or any other static host — usually in under an hour. The same source code works everywhere.

I'm not technical. Can I actually update the site myself?

Yes, and the handover is designed for exactly that. The owner's manual is written in plain English, specific to your site, and paired with a 30-minute walkthrough recording. For clients who'd rather not touch it, an optional Decap CMS integration adds a simple non-technical editor, or you can take a small retainer for routine updates.

How is this different from a freelancer doing the same thing?

The handover packet. Any developer can build a static site on GitHub Pages. What you get with GitFoundry is the deliverable that makes the site genuinely yours: a manual you can read, a walkthrough you can watch, a runbook for emergencies, a migration playbook, and a real support window after launch.

What if GitFoundry stops trading?

Your site keeps working. Your repository is already in your GitHub account, your domain is registered in your name, and your hosting is on a Microsoft-owned platform that costs nothing. There is no dependency on GitFoundry continuing to exist.

Are you VAT-registered?

Yes — all displayed prices include UK VAT. No surprise figures at the invoice stage.
